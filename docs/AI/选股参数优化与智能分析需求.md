# 选股参数优化与智能分析系统需求文档

## 一、需求概述

构建一个智能化的选股参数优化与分析系统，通过参数调整、效果对比、自动分析总结，形成私有知识库，持续优化选股策略。

---

## 二、核心功能模块

### 2.1 选股参数配置模块

#### 功能描述
- 支持手动调整选股参数（如涨幅阈值、成交量、市值等）
- 参数配置可保存为不同的策略方案
- 支持参数版本管理和历史回溯

#### 参数示例
```yaml
策略名称: 次新股激进策略
参数配置:
  - 涨幅范围: 3% → 5%
  - 上市时间: 近一年
  - 成交量: > 1亿
  - 市值范围: 50亿-200亿
```

#### 技术实现
- 参数配置存储：JSON/YAML 格式
- 版本管理：Git 或数据库版本表
- 配置界面：Web 表单 + 预设模板

---

### 2.2 参数对比分析模块

#### 功能描述
- 对比不同参数配置下的选股结果
- 分析哪些股票表现更好，哪些表现下降
- 生成对比报告（涨幅、收益率、胜率等指标）

#### 对比维度
- **股票级别对比**：同一只股票在不同参数下的表现
- **策略级别对比**：不同策略的整体收益对比
- **时间维度对比**：同一策略在不同时间段的表现

#### 输出示例
```
参数调整：涨幅 3% → 5%
测试周期：2025-01-01 至 2025-12-31

表现提升的股票（10只）：
- SH600000 浦发银行：收益率 +15% → +22% ↑
- SZ000001 平安银行：收益率 +8% → +18% ↑

表现下降的股票（5只）：
- SH600036 招商银行：收益率 +12% → +5% ↓

整体指标：
- 平均收益率：+10.5% → +12.3% ↑
- 胜率：65% → 70% ↑
- 最大回撤：-8% → -6% ↓
```

### 2.3 模块化语言体系

#### 功能描述
- 将选股逻辑封装为可复用的模块
- 通过关键词或菜单快速启动不同策略
- 支持模块组合和自定义

#### 预设模块示例
1. **次新股模块**：筛选上市时间 < 1年的股票
2. **涨幅模块**：筛选涨幅在指定区间的股票
3. **成交量模块**：筛选成交量 > 阈值的股票
4. **技术指标模块**：MACD、KDJ、RSI 等技术分析
5. **财务指标模块**：PE、PB、ROE 等基本面分析
6. **行业板块模块**：按行业分类筛选
7. **资金流向模块**：主力资金净流入分析

#### 使用方式
- **关键词启动**：输入 "次新股 + 涨幅5% + 成交量过亿"
- **菜单点击**：Web 界面勾选模块并配置参数
- **API 调用**：`/api/strategy/run?modules=cxg,zf,cjl&params=...`

### 2.4 每日自动分析总结

#### 功能描述
- 每日收盘后自动分析当日交易执行情况
- 总结选股策略的表现（命中率、收益率）
- 自动生成优化建议

#### 分析内容
1. **交易执行情况**
   - 今日选出股票数量
   - 触发买入/卖出信号的股票
   - 实际涨跌幅统计

2. **策略表现评估**
   - 命中率：选出的股票中上涨的比例
   - 平均涨幅：所有选出股票的平均涨幅
   - 最佳/最差股票

3. **优化建议**
   - 参数调整建议：如 "建议将涨幅阈值从 3% 调整为 4%"
   - 模块组合建议：如 "建议增加成交量过滤条件"
   - 风险提示：如 "今日大盘下跌 2%，建议降低仓位"

#### 输出格式
```markdown
# 每日选股分析报告 - 2026-03-18

## 一、交易执行情况
- 选出股票：15 只
- 触发买入信号：8 只
- 触发卖出信号：3 只

## 二、策略表现
- 命中率：73.3% (11/15 上涨)
- 平均涨幅：+2.8%
- 最佳股票：SH600000 (+8.5%)
- 最差股票：SZ000001 (-1.2%)

## 三、优化建议
1. 涨幅阈值建议从 3% 调整为 4%，可提升命中率至 80%
2. 建议增加 "成交量 > 2亿" 过滤条件，减少低流动性股票
3. 今日大盘震荡，建议关注防御性板块

## 四、风险提示
- 次新股板块波动较大，建议控制单只股票仓位 < 5%
```

### 2.5 周线/月线/季度/年度总结

#### 功能描述
- 自动生成不同时间维度的策略总结报告
- 分析长期趋势和周期性规律
- 对比不同时间段的策略表现

#### 报告内容

**周线总结（每周一生成）**
- 本周选股数量、命中率、收益率
- 本周最佳/最差策略
- 下周策略调整建议

**月线总结（每月1号生成）**
- 本月整体表现统计
- 月度最佳股票 TOP 10
- 月度策略优化记录

**季度总结（每季度末生成）**
- 季度收益曲线
- 不同市场环境下的策略表现
- 季度参数调整历史

**年度总结（每年末生成）**
- 年度收益率、最大回撤、夏普比率
- 年度最佳策略组合
- 年度优化路径回顾

#### 数据可视化
- 收益曲线图
- 命中率趋势图
- 参数调整时间线
- 策略对比雷达图

## 三、技术实现方案

### 3.1 技术栈
- **后端**：Java 8 + Spring Boot
- **数据库**：MySQL（存储配置和报告）+ Redis（缓存实时数据）
- **任务调度**：Spring Scheduler / Quartz
- **数据分析**：Python（可选，用于复杂统计分析）
- **前端**：Vue.js / React（参数配置界面）
- **文档存储**：Markdown 文件 + Git 版本管理

### 3.2 核心模块设计

#### 参数配置服务
```java
@Service
public class StrategyConfigService {
    // 保存策略配置
    void saveStrategy(StrategyConfig config);

    // 加载策略配置
    StrategyConfig loadStrategy(String strategyName);

    // 对比两个策略
    ComparisonReport compareStrategies(String strategy1, String strategy2);
}
```

#### 自动分析服务
```java
@Service
public class AutoAnalysisService {
    // 每日分析
    @Scheduled(cron = "0 0 16 * * ?") // 每天16:00执行
    void dailyAnalysis();

    // 周线分析
    @Scheduled(cron = "0 0 9 ? * MON") // 每周一9:00执行
    void weeklyAnalysis();

    // 生成优化建议
    List<Suggestion> generateSuggestions(AnalysisResult result);
}
```

#### 知识库服务
```java
@Service
public class KnowledgeBaseService {
    // 存档报告
    void archiveReport(Report report, ReportType type);

    // 检索报告
    List<Report> searchReports(SearchCriteria criteria);

    // 获取历史策略
    List<StrategyConfig> getStrategyHistory(String strategyName);
}
```

### 3.3 数据库设计

#### 策略配置表
```sql
CREATE TABLE strategy_config (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    strategy_name VARCHAR(100) NOT NULL,
    version VARCHAR(20) NOT NULL,
    parameters JSON NOT NULL,
    create_time DATETIME NOT NULL,
    creator VARCHAR(50),
    description TEXT,
    UNIQUE KEY uk_name_version (strategy_name, version)
);
```

#### 分析报告表
```sql
CREATE TABLE analysis_report (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    report_type ENUM('DAILY', 'WEEKLY', 'MONTHLY', 'QUARTERLY', 'YEARLY'),
    report_date DATE NOT NULL,
    strategy_name VARCHAR(100),
    hit_rate DECIMAL(5,2),
    avg_return DECIMAL(10,2),
    suggestions JSON,
    content TEXT,
    create_time DATETIME NOT NULL,
    INDEX idx_date_type (report_date, report_type)
);
```

#### 优化建议表
```sql
CREATE TABLE optimization_suggestion (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    suggestion_type VARCHAR(50),
    content TEXT NOT NULL,
    confidence_score DECIMAL(3,2),
    status ENUM('PENDING', 'APPLIED', 'REJECTED'),
    create_time DATETIME NOT NULL,
    apply_time DATETIME,
    INDEX idx_status (status)
);
```

## 五、预期效果

1. **效率提升**：参数调整和效果对比从手动 1 小时缩短到自动 5 分钟
2. **决策支持**：每日自动生成优化建议，减少主观判断误差
3. **知识沉淀**：形成私有知识库，积累选股经验和市场规律
4. **持续优化**：通过长期数据分析，不断迭代优化策略参数
5. **风险控制**：及时发现策略失效信号，降低投资风险

---

## 六、注意事项

1. **数据准确性**：确保实时数据和历史数据的准确性
2. **性能优化**：大量历史数据分析需要优化查询性能
3. **安全性**：知识库数据需要加密存储，防止泄露
4. **可扩展性**：模块化设计，便于后续增加新的分析维度
5. **用户体验**：界面简洁易用，报告清晰易懂

---

## 七、后续扩展方向

1. **AI 辅助决策**：引入机器学习模型，自动发现最优参数组合
2. **多策略组合**：支持多个策略并行运行，分散风险
3. **实时预警**：当策略表现异常时，实时推送预警通知
4. **社区分享**：（可选）匿名分享策略表现,学习他人经验
5. **回测系统**：基于历史数据进行策略回测验证
